# Team Task Manager

A modern, full-stack web application for collaborative project and task management. Built with React, Node.js, PostgreSQL, and deployed on Railway.

## 🌟 Features

- **User Authentication** - Secure JWT-based authentication with email/password signup and login
- **Project Management** - Create, organize, and manage projects with team members
- **Task Management** - Create, assign, and track tasks with status and priority levels
- **Team Collaboration** - Invite team members, assign roles (admin/member), and collaborate in real-time
- **Dashboard** - Comprehensive overview with stats, task breakdown, and recent activities
- **Task Filtering** - Advanced filtering by status, priority, assignee, and due date
- **Responsive Design** - Mobile-friendly interface built with Tailwind CSS
- **Real-time Updates** - React Query for efficient data synchronization
- **Error Handling** - Global error handling with user-friendly error messages
- **Rate Limiting** - Protect auth endpoints from brute force attacks

## 🚀 Quick Start

### Prerequisites

- Node.js (v18+)
- PostgreSQL (v14+)
- npm or yarn

### Local Setup

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd team-task-manager
   ```

2. **Setup Backend**
   ```bash
   cd backend
   npm install
   
   # Create .env file
   cat > .env << EOF
   DATABASE_URL=postgresql://user:password@localhost:5432/team_task_manager
   JWT_SECRET=your-super-secret-jwt-key
   JWT_EXPIRE=7d
   NODE_ENV=development
   PORT=5000
   EOF
   
   # Run migrations
   npm run migrate
   
   # Start server
   npm run dev
   ```

3. **Setup Frontend**
   ```bash
   cd ../frontend
   npm install
   
   # Create .env file
   cat > .env << EOF
   VITE_API_URL=http://localhost:5000/api
   EOF
   
   # Start dev server
   npm run dev
   ```

4. **Open browser**
   ```
   http://localhost:5173
   ```

## 🚢 Deployment on Railway

### 1. Create Railway Account
- Go to [railway.app](https://railway.app)
- Sign up with GitHub

### 2. Create New Project
- Click "New Project" → "Deploy from GitHub repo"
- Select your repository

### 3. Configure Backend Service

```bash
# In Railway dashboard:
# Environment Variables for backend:
DATABASE_URL=<auto-generated by Railway PostgreSQL plugin>
JWT_SECRET=<generate-strong-secret>
JWT_EXPIRE=7d
NODE_ENV=production
PORT=8080
```

**Add PostgreSQL Plugin:**
- Click "Add Plugin" → Select "PostgreSQL"
- Railway will auto-populate `DATABASE_URL`

**Configure Start Script:**
```json
{
  "scripts": {
    "start": "node src/server.js",
    "dev": "nodemon src/server.js"
  }
}
```

### 4. Configure Frontend Service

Create `railway.json` in frontend root:
```json
{
  "build": {
    "builder": "nixpacks",
    "buildCommand": "npm run build"
  }
}
```

**Environment Variables:**
```
VITE_API_URL=https://your-railway-backend-domain.up.railway.app/api
```

### 5. Deploy

```bash
# Install Railway CLI
npm i -g @railway/cli

# Login
railway login

# Link project
railway link

# Deploy
railway up
```

Your app will be live at: `https://<project-name>.up.railway.app`

## 🐛 Troubleshooting

### Database Connection Error
```bash
# Check PostgreSQL is running
psql -U postgres -d team_task_manager

# Reset database
npm run migrate:reset
```

### JWT Token Expired
- Automatically handled by refresh token mechanism
- User will be redirected to login

### CORS Issues
- Ensure backend `CORS_ORIGIN` matches frontend URL
- Check `.env` configuration
